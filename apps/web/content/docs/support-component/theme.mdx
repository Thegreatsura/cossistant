## Philosophy
The Support widget borrows your site colors whenever it can. It only exposes variables that start with `co-`, so the rest of your Tailwind or Shadcn setup stays untouched.

## How we pick a value
For every style variable the widget checks three places, always in the same order:

1. A widget-only override you place on `.cossistant` (`--co-theme-*`).
2. The host app token, such as Shadcn's `--background`, `--radius`, or `--primary`.
3. A baked-in default from Cossistant so the widget always renders sensibly.

| Final widget variable | Step 1: widget override (`--co-theme-*`) | Step 2: site variable | Step 3: built-in default |
| --- | --- | --- | --- |
| `--co-radius` | `--co-theme-radius` | `--radius` | `0.625rem` |
| `--co-background` | `--co-theme-background` | `--background` | Soft white (`oklch(99% 0 0)`) |
| `--co-foreground` | `--co-theme-foreground` | `--foreground` | Deep gray (`oklch(14.5% 0 0)`) |
| `--co-border` | `--co-theme-border` | `--border` | Light gray (`oklch(92.2% 0 0)`) |
| `--co-accent` | `--co-theme-accent` | `--accent` or `--primary` | Indigo (`oklch(20.5% 0 0)`) |
| `--co-ring` | `--co-theme-ring` | `--ring` or `--primary` | Same indigo |
| `--co-muted` | `--co-theme-muted` | `--muted` | Blend of background + foreground |
| `--co-muted-foreground` | `--co-theme-muted-foreground` | `--muted-foreground` | Softer text blend |

Background shades (`--co-background-50` -> `--co-background-600`) are created with `color-mix`, gently blending the base background with the text color. If a browser cannot do `color-mix`, we fall back to safe neutral values.

## Light and dark mode
The widget follows the mode your site is in. If your layout toggles a `.dark` class we adapt automatically. You can also force the dark palette by adding `data-color-scheme="dark"` to the `.cossistant` wrapper. The three-step lookup stays the sameâ€”we only swap in darker defaults at Step 3 when needed.

## Overriding Styles
Apply overrides on the widget container. Host tokens continue to win, but any `--co-theme-*` variable lets you fine-tune the experience without touching global CSS:

```css
.cossistant {
	--co-theme-background: oklch(0.96 0.02 140);
	--co-theme-accent: var(--brand-primary);
	--co-theme-radius: 12px;
}
```

If your app does not expose Shadcn tokens, set the `co-theme` variables anywhere in your stylesheet:

```css
:root {
	--co-theme-background: #0f172a;
	--co-theme-foreground: #f1f5f9;
	--co-theme-accent: #38bdf8;
	--co-theme-accent-foreground: #0b1120;
}
```
